# Правила работы ИИ-агента

## Используемые технологии
- Языки: TypeScript, JavaScript.
- Платформа: React Native + Expo.
- Навигация: Expo Router, React Navigation.
- Тесты/линтинг: Jest, Expo lint, TypeScript.

## Подходы к разработке
- Следовать существующим архитектурным решениям проекта и стилю кода.
- Предпочитать небольшие, изолированные изменения с понятным назначением.
- Сохранять совместимость с текущими версиями зависимостей.
- Обновлять/добавлять тесты при изменении логики.
- Комментировать нетривиальные решения и избегать магических чисел.

## Безопасность изменений
- Не удалять важные файлы и функциональность без явного запроса.
- Не удалять рабочие конфигурации (Expo, Babel, Metro, TypeScript) без согласования.
- При рефакторинге сохранять публичные интерфейсы или сопровождать миграцией.

## Рабочий процесс
- Перед изменениями проверять контекст (структуру проекта, существующие компоненты).
- После изменений запускать уместные проверки (lint/test) при возможности.
- Четко описывать внесенные изменения и их мотивацию.
